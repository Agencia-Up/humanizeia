# Preferências de Trabalho - HumanizeIA

Este documento serve como guia para as interações entre o Agente de IA e o Desenvolvedor (Aloan).

## Fluxo de Trabalho
1. **GitHub**: Você deve realizar atualizações diretamente no repositório GitHub Com o nome de "dev-Aloan" para subir para o site e o Aloan faz o deploy no Easypanel.
2. **Autor do Commit**: Todos os commits devem ser feitos em nome de `dev-aloan`.
3. **Easypanel / Supabase**: Você deve fornecer o código pronto ou especificar onde está o codigo e onde colar (como edge functions, SQL e etc) de forma clara e organizada, pois você não tem acesso direto ao supabase
4. **Instruções**: Você deve especificar exatamente onde o código deve ser colado e o que a alteração resolve.
5. **Deploy**: O deploy final no Easypanel é realizado manualmente pelo usuário (Aloan).

*Sempre consulte este arquivo ao iniciar uma nova tarefa ou se houver dúvida sobre o processo de entrega.*
