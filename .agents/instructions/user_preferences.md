# Preferências de Trabalho - HumanizeIA

Este documento serve como guia para as interações entre o Agente de IA e o Desenvolvedor (Aloã).

## Fluxo de Trabalho
1. **GitHub**: O Agente de IA deve realizar commits diretamente no repositório GitHub.
2. **Autor do Commit**: Todos os commits devem ser feitos em nome de `dev-aloan`.
3. **Easypanel / Supabase**: O Agente de IA deve fornecer o código pronto para ser colado nas interfaces (como Dashboard do Supabase ou Easypanel) de forma clara e organizada.
4. **Instruções**: O Agente deve especificar exatamente onde o código deve ser colado e o que a alteração resolve.
5. **Deploy**: O deploy final no Easypanel é realizado manualmente pelo usuário (Aloã).

---
*Sempre consulte este arquivo ao iniciar uma nova tarefa ou se houver dúvida sobre o processo de entrega.*
