## UazapiWhatsAppSender

Criado `UazapiWhatsAppSender implements WhatsAppSendPort`.

Caracteristicas:

1. Base URL obrigatoriamente HTTPS.
2. Host precisa estar em allowlist.
3. URL com usuario/senha/query/hash e rejeitada.
4. Destino e normalizado para telefone BR (`55...`) como no v2.
5. Texto usa endpoints compativeis com v2:
   - `/send/text` com `number`
   - fallback `/send/text` com `remoteJid`
   - fallback `/message/sendText/{instanceName}` quando ha instanceName
6. Midia usa `/send/media` com `type: "image"`.
7. Midia exige URL HTTPS.
8. Token e resolvido no envio, nunca no config/estado/outbox.
9. Erros HTTP/transporte sao sanitizados; corpo remoto com token/URL nao volta no erro.
10. HTTP OK da Uazapi vira **receipt `accepted`**, nunca `delivered`.

## Decisao de receipt

Mantida a decisao conservadora: a Uazapi, pelo comportamento conhecido no v2, confirma aceite HTTP, nao entrega real ao lead. Portanto o sender retorna `accepted`.

Consequencia correta: efeitos criticos com `onSuccess` nao atualizam memoria conversacional ate existir callback/reconciliacao de `delivered` ou uma politica explicita de risco aprovada pelo dono.
