## Invariante

O piloto v3 so pode casar com os dois IDs exatos:

- tenant/user_id: `ecb26258-ffe6-4fe2-9efc-8ab2fc3a61b0`
- agent_id: `d4fd5c38-dd37-4da5-a971-5a7b7dfb9185`

Nao autorizar por email, nome do agente, instance_id, agent fallback ou primeiro ativo. Tenant certo com agent errado bloqueia. Agent certo com tenant errado bloqueia. Modo invalido vira `off`.

## Modos

`PEDRO_V3_PILOT_MODE` aceita:

- `off` (default): nenhum acesso v3.
- `shadow`: autorizado somente para o par exato.
- `active`: autorizado somente para o par exato, mas o handler ativo ainda NAO esta ligado; o webhook loga e cai com seguranca para Pedro v2.

Esse fallback para v2 e proposital nesta fase para nao deixar lead sem resposta enquanto os dispatchers ativos do v3 nao forem implementados.
