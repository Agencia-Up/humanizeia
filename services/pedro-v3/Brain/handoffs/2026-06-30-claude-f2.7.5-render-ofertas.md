- "Bom dia" -> "Temos disponivel este modelo aqui para voce:**ONIX2014Ele** esta com 132.623 km..."

Raiz: `ResponseRenderer` fazia `out += v.modelo` / `out += v.ano.toString()` / `out += formatCurrency(...)` **sem separador**. O LLM emitia, por veiculo, uma sequencia de `vehicle_ref(modelo)` + `vehicle_ref(ano)` + `money_ref` -> tudo grudava. Roteamento, memoria e grounding ja OK (Fase 1) — era so apresentacao.

## Correcao — deterministica, sem if por marca/modelo
1. **Nova parte `vehicle_offer_list`** (`domain/decision.ts`): `{ type, vehicleKeys: string[] }`. O modelo so escolhe QUAIS veiculos; a formatacao e do sistema.
2. **Helper PURO `engine/vehicle-offer-render.ts`** (`renderVehicleOfferList(vehicles, {maxItems})`):
   - lista numerada `1. Marca Modelo Ano — R$ x`; detalhe indentado `   km · cambio · cor`;
   - `formatBRL` (pt-BR), `formatKm` (separador BR);
   - campos ausentes omitidos sem buraco (sem `undefined`, sem ` · ` solto, sem linha de detalhe vazia);
   - **preco 0/ausente -> "preço a confirmar"** (nunca "R$ 0"); **ano ausente nao inventado**;
   - limite default **5** (configuravel).
3. **`ResponseRenderer` reescrito em SEGMENTOS** (`engine/response-renderer.ts`):
   - junta partes com SEPARADOR (`needsSpace`): nunca cola dois tokens palavra/numero/pontuacao ("voce:ONIX" -> "voce: ONIX"; refs adjacentes "Onix" + "2014" -> "Onix 2014");
   - `vehicle_offer_list` vira BLOCO separado por linha em branco (intro acima, pergunta abaixo);
   - grounding mantido: chave do offer_list/ vehicle_ref / money_ref ausente dos fatos -> **falha fechada** (throw -> deny -> retry/terminal-safe).
4. **Grounding (`engine/policy-engine.ts`):** `validateResponse` valida cada chave do `vehicle_offer_list` contra o catalogo do tenant (defesa em profundidade); o texto renderizado segue checado por marca/modelo/preco vs QueryResults (passos 2/3 existentes).
5. **Prompt compose (`adapters/llm/openai-chat-model.ts`):** apresentar estoque **so via `vehicle_offer_list`** (ate 5 chaves); intro curto (text) ANTES, UMA pergunta de funil (text) DEPOIS; nao escrever veiculos/precos/anos/lista a mao.
6. **`VehicleFact` (`domain/types.ts`):** `cambio?`/`cor?` opcionais — renderizados "se houver"; nenhuma fonte preenche ainda (additivo, sem efeito agora).
7. **Decoder (`adapters/llm/prompt-bound-conversation.ts`):** `validResponsePart` aceita `vehicle_offer_list` com `vehicleKeys` nao-vazio.
