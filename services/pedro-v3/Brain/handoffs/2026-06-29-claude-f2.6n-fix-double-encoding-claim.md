# Handoff - F2.6N - ROOT CAUSE do "v3 nao responde": double-encoding no filtro PostgREST

Data: 2026-06-29
Executor: Claude (auditor: Codex)
Escopo: Pedro v3 `Agent/` (gateway + teste). **Sem deploy manual, sem banco, sem ativacao.**

## Como cheguei na raiz

F2.6L/M tornaram a falha visivel: o `v3_inbox.last_error` do evento que falhou gravou
**`Error: claimed inbox record missing`** (vem de `conversation-engine.ts:128`).

Esse erro acontece quando `claimBurst` claima N eventos mas `persistence.get(eventId)` devolve `null`
para algum deles. Investiguei e a causa e um **DOUBLE-ENCODING** no `SupabaseServiceGateway`:

- `get()` filtra `v3_inbox` por `event_id`. O `event_id` real e `uazapi:<hash>` (com `:`).
- `encodeFilter` fazia `eq.${encodeURIComponent(value)}` -> `eq.uazapi%3A<hash>`.
- Esse valor entra num `URLSearchParams`, cujo `toString()` **re-encoda o `%`** -> `eq.uazapi%253A<hash>`.
- O PostgREST decodifica uma vez -> procura `event_id = "uazapi%3A<hash>"` (literal, com `%3A`), que
  **nao casa** com `"uazapi:<hash>"` -> retorna `[]` -> `get()=null` -> "claimed inbox record missing"
  -> turno falha **toda vez** -> nenhuma resposta -> fallback pro v2 (o print do dono).

Por que passou despercebido: os RPCs (claim/ingest/commit) mandam args no BODY JSON (sem esse encoding),
e os testes do adapter usavam ids sem `:` / transporte fake. O bug so aparece com PostgREST real + id com `:`.
(E por isso o `v3_record_inbox_error`, que e RPC, gravou o erro certinho enquanto o `get()` quebrava.)

## Fix

`Agent/src/runtime/supabase-service-gateway.ts` -> `encodeFilter` agora retorna `eq.${String(value)}` (valor
CRU). O `URLSearchParams.toString()` faz o encoding UMA vez so, correto. Vale para `selectOne`/`selectMany`/
`count` (todos usam `URLSearchParams`). RPCs nao mudam (body JSON).

## Teste novo (pega a regressao)

`Agent/tests/run-gateway-filter.ts` (5 checks): transporte fake captura a URL e prova que `event_id`/
`conversation_id` com `:` saem **single-encoded** (`uazapi%3A...`, sem `%253A`), uuid intacto, valor simples
(`status=eq.pending`) intacto.
