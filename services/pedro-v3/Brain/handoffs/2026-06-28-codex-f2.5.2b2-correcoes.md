## Correcoes feitas

1. Segredo local:
   - `Agent/scratch/inspect_real_sources.mjs` foi redigido.

2. Catalogo dinamico:
   - `isVehicleKeyInCatalog` aceita `vehicleKey` factual do estoque por match exato (`provider:id`).
   - Mantida compatibilidade com chave legada `brand|model|year` usada em testes antigos.

3. Cache read-side:
   - `ReadCache` agora usa `version` + `flightId` por voo pendente.
   - `invalidate` remove pendentes e impede que promessa antiga repovoe cache ou apague voo novo.
   - Cobertura nova para single-flight, invalidacao concorrente, TTL e LRU.

4. Decoder de estoque:
   - `decodeNormalizedVehicle` falha fechado para objeto/array em campos escalares.
   - `vehicle_id`/`vehicleExternalKey` so aceitam string/number.
   - Marca/modelo obrigatorios so aceitam string.
   - Numeros brasileiros como `R$ 70.990,00` e `92.900` sao parseados corretamente.
   - `parseVehiclePhotos` ignora URL nao-string, HTTP e host invalido; `generatePhotoId` rejeita URL invalida.

5. Loader de estoque:
   - `V2StockLoader` valida agente dono e ativo via `getOwnedAgent(ref)` antes de listar integracoes/resolver credencial.
   - Metadata duplicada, provider invalido ou tenant divergente falha fechado.
   - Sem agente dono: nao resolve credencial e nao faz fetch.
